# Contributing to Awesome MCP Resources

Thank you for your interest in contributing! This document provides guidelines for contributing to this resource list.

## How to Contribute

1. **Fork the repository**
2. **Create a new branch** for your contribution
3. **Make your changes**
4. **Submit a pull request**

## What to Contribute

We welcome:

- New MCP servers (official or community)
- Tutorials and courses
- Video resources
- Example projects
- Tools and libraries
- Blog posts and articles
- Bug fixes and improvements to existing content

## Contribution Guidelines

### Adding a Resource

When adding a new resource, please ensure:

1. **It's MCP-related** - Directly related to Model Context Protocol
2. **It's accessible** - Links work and content is publicly available
3. **It's documented** - Has clear description and usage instructions
4. **It's maintained** - Recently updated or actively maintained
5. **It adds value** - Provides unique or useful information for learners

### Format

Please follow the existing format:

- Use clear, descriptive titles
- Provide accurate descriptions
- Include relevant links
- Add to the appropriate section
- Maintain alphabetical order where applicable

### Quality Standards

- Check for spelling and grammar
- Verify all links are working
- Ensure code examples are tested
- Provide context and explanation

## Pull Request Process

1. Update the README.md with your additions
2. Follow the existing structure and formatting
3. Provide a clear description of what you're adding
4. Reference any related issues
5. Wait for review and feedback

## Code of Conduct

- Be respectful and constructive
- Welcome newcomers
- Focus on helping others learn
- Credit original authors

## Questions?

Open an issue or join the discussion in the MCP Discord community.

Thank you for helping make MCP more accessible to everyone!
